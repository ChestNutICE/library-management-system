# 多台电脑联网版升级方案

当前程序使用本机 SQLite。不能把 `library.db` 放进共享文件夹让多台电脑同时写入，这会造成锁冲突甚至数据库损坏。

## 安全架构

```text
多台 PySide6 客户端 → HTTPS API 服务 → PostgreSQL/MySQL
```

## 已建立的迁移边界

- 界面只调用 `services` 业务层，不直接持有数据库连接。
- 权限、日志、借还规则都集中在服务层，可迁移为 API 端点。
- Excel、PDF、备份等功能与界面解耦。

## 正式联网版需要增加

1. FastAPI 服务端与 PostgreSQL 数据库。
2. 登录令牌、HTTPS、密码重试限制和服务端权限校验。
3. 数据库迁移工具和 SQLite 数据导入程序。
4. 客户端网络异常、超时、重试与离线提示。
5. 服务器部署、域名/证书、备份和监控。

联网部署涉及服务器地址、数据库账号、证书和网络开放范围，必须由部署者提供，不能在单机包中写死。当前版本继续安全地以单机模式运行。
