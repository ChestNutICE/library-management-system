# 图书管理系统第一天需求基线

## 项目背景

图书资料、读者信息和借阅记录如果依赖纸质登记或零散电子表格，容易出现查询缓慢、库存不一致、借阅历史难追踪等问题。本项目开发一个 Windows 单机桌面图书管理系统，使用 SQLite 保存数据，为课程设计提供完整、可演示的软件工程成果。

## 用户角色

### 管理员

- 维护图书和分类资料。
- 维护读者账户。
- 办理借书与还书。
- 查询借阅、归还和逾期记录。
- 查看基本统计信息。

### 读者

- 查询可借图书。
- 查看个人借阅记录。
- 修改个人密码。

## 核心功能

1. 登录和角色权限。
2. 图书新增、修改、查询和停用。
3. 读者新增、修改、查询和停用。
4. 借书、还书和库存同步。
5. 借阅记录、逾期记录和基本统计。

## 冻结业务规则

- 默认借期为 30 天。
- 每名读者最多同时借阅 5 本书。
- 可借库存为 0 时不能借书。
- 同一读者不能重复借阅尚未归还的同一本书。
- 存在历史记录的图书和读者只允许停用，不物理删除。
- 借书和还书涉及的多项数据库修改必须位于同一事务中。

## 本次不做

- 网页版和手机 App。
- 多台电脑联网和云端数据库。
- 短信、邮件、摄像头扫码和在线支付。
- 复杂自定义权限系统。

## 第一阶段验收标准

- 开发环境和虚拟环境可用。
- PySide6 能创建窗口。
- 项目目录结构清晰。
- SQLite 能自动创建五张核心数据表。
- 默认管理员和六个分类只初始化一次。
- 图书新增、查询、修改、停用和重复 ISBN 拦截测试通过。
- 程序关闭并重新连接数据库后，数据仍然存在。
